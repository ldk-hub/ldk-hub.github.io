안녕하세요. 백엔드 시스템 설계부터 프론트엔드 사용자 경험까지, 전체 라이프사이클을 고민하는 풀스택 엔지니어 이동옥입니다.

개발 블로그를 운영하고 수많은 개인/실무 프로젝트를 진행해 오면서, 제 스스로 어떤 길을 걸어왔고 어떤 기술적 철학을 가지게 되었는지 한 번쯤 총결산이 필요하다고 느꼈습니다. 이 글은 제 **과거의 개발 여정에 대한 회고록이자, 기업 제출용 포트폴리오의 요약본**입니다.

## 💡 개발 철학 (My Philosophy)

소프트웨어 개발에 있어 제가 가장 중요하게 생각하는 세 가지 원칙은 다음과 같습니다.

1. **테스트 주도 개발 (TDD)과 클린 코드**
   기능을 빨리 만들어 내는 것보다, '견고하게' 만드는 것을 선호합니다. TDD는 단순히 버그를 줄이는 도구가 아니라, 응집도가 높고 결합도가 낮은 아키텍처를 강제하는 가장 훌륭한 나침반이라고 생각합니다.
2. **트래픽을 버티는 지속 가능한 아키텍처 (MSA & DDD)**
   초기 모놀리식 구조가 가지는 생산성의 장점을 이해하지만, 비즈니스가 성장함에 따라 겪게 되는 병목 현상을 `Spring Cloud`, `Kafka`, `Kubernetes`를 활용한 MSA 환경으로 풀어내는 것에 큰 흥미를 느낍니다.
3. **사용자 경험(UX) 극대화**
   백엔드 개발자의 역할이 단순히 API를 내려주는 데서 끝난다고 생각하지 않습니다. 최종 사용자에게 어떻게 렌더링될지, 프론트엔드의 성능 부하는 어떻게 줄일지(예: 캐싱, 최적화된 쿼리) 고민하는 것이 진정한 엔지니어의 자세라고 믿습니다.

## 🚀 대표 프로젝트 하이라이트

그동안 진행했던 수많은 프로젝트 중, 기술적으로 가장 깊게 고민했던 핵심 프로젝트들을 소개합니다.

### 1. AI 가상 오피스 모니터링 시스템 (2D Pixel RPG)
*단순한 텍스트 기반 대시보드를 넘어선 혁신적인 모니터링 뷰어*

- **스택**: React, Zustand, Konva.js, Tailwind CSS, TypeScript
- **핵심 경험**:
  - HTML DOM이 아닌 `Canvas API(Konva.js)`를 사용하여 **60FPS의 부드러운 렌더링 최적화** 달성.
  - 16-bit 픽셀 아트와 Y-Sorting(깊이 정렬) 알고리즘을 직접 구현하여, 에이전트들이 겹칠 때의 시각적 오류(Depth issue) 완벽 해결.
  - FSM(유한 상태 기계) 아키텍처를 프론트엔드에 적용하여, 서버에서 푸시되는 AI 상태(Working, Idle)에 따라 자율적으로 애니메이션이 전환되도록 설계.

### 2. 대용량 트래픽 대응을 위한 MSA 전환
*모놀리식의 한계를 부수고 서비스 안정성을 높이다*

- **스택**: Spring Boot, Spring Cloud Config, Kafka, Docker
- **핵심 경험**:
  - `Kafka` 기반의 비동기 메시징 시스템을 도입하여 서비스 간 강결합을 끊고 트랜잭션 장애 격리(Isolation) 실현.
  - 마이크로서비스들의 환경설정을 `Spring Cloud Config`로 중앙 통제하여 배포 파이프라인의 효율성 극대화.

### 3. 고성능 보안 API 서버 아키텍처
*인증, 인가, 그리고 데이터베이스 성능 튜닝의 정석*

- **스택**: Spring Boot, Spring Security (OAuth2), JPA(Hibernate), Redis
- **핵심 경험**:
  - JWT 기반의 글로벌 Stateless 인증/인가 아키텍처 구현.
  - ORM 환경에서 가장 흔히 발생하는 `N+1` 문제를 `Fetch Join`과 `BatchSize` 전략으로 해결하여 쿼리 실행 횟수 80% 이상 단축.
  - `Redis` 인메모리 캐싱을 통한 읽기 부하 감소 및 글로벌 세션 공유 처리.
